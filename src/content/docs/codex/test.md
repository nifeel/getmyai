## 为什么要做模型测试

安装命令执行成功，不代表模型调用链路已经可用。测试页应该负责验证：

- Key 是否生效
- 模型名是否正确
- 流式输出是否稳定
- 网络与代理是否影响返回

## 最小验证步骤

1. 启动 `codex`
2. 选择目标模型
3. 发送一条简单提示词
4. 观察是否正常返回结果

## 建议的检查点

- 返回是否稳定
- 是否有 `401` 或 `429`
- 首字节延迟是否异常
- 长回复时是否出现中断

## 为什么单独一页更合适

测试流程会随着模型和客户端增加而扩展，单独成页后，你后面可以继续补：

- 不同模型推荐测试词
- 图像或多模态测试
- 代理环境测试
- 性能基线记录
